Confirming or rejecting predictions

GitHub Pages serves static files only -- there's no server on the other end of this page to accept a write, so a "Confirm" click here has nowhere to go. Confirmation happens locally instead, with python -m pipeline.review.

  1. In your local checkout, make sure data/aggregator.db is the current archive -- it isn't committed to git, so if you don't already have a synced copy, grab the latest backup:
    gh release download db-latest --repo oisinbogue/Economic-News-Aggregator --pattern "aggregator.db.*.gz" --output data/aggregator.db.gz && gunzip data/aggregator.db.gz
  2. Run python -m pipeline.review.
  3. For each pending prediction you'll see the claim, the proposed verdict, the LLM's reasoning, and any cited articles, then choose: y confirm, n reject (rechecked later), x give up (expires, no verdict), s skip, q quit.
  4. Confirmed verdicts show up here (and count toward the leaderboards below) after the next site rebuild -- automatic within a few hours on the scheduled pipeline, or trigger it immediately from the repo's Actions tab.
